# APP_FLOW - 应用流程文档

## 1. 系统启动流程

```
用户启动应用
    ↓
加载配置文件
    ↓
初始化数据库连接
    ↓
检查数据库是否存在
    ├─ 否 → 创建数据库和表结构
    └─ 是 → 连接现有数据库
    ↓
显示首页（板块总览）
```

## 2. 数据更新流程

### 2.1 完整数据更新（首次使用）

```
用户点击"完整数据更新"
    ↓
进度提示：开始获取数据...
    ↓
【步骤1】获取板块列表
    ├─ 调用 akshare 获取行业板块列表
    ├─ 调用 akshare 获取概念板块列表
    ├─ 存储到 sector 表
    └─ 进度：10%
    ↓
【步骤2】获取板块成分股
    ├─ 遍历每个板块
    ├─ 调用 akshare 获取板块成分股列表
    ├─ 存储到 sector_stocks 表
    └─ 进度：20%
    ↓
【步骤3】获取股票基本信息
    ├─ 遍历所有股票
    ├─ 调用 akshare 获取股票基本信息
    ├─ 存储到 stocks 表
    └─ 进度：30%
    ↓
【步骤4】获取7年历史数据
    ├─ 遍历所有股票
    ├─ 对每只股票：
    │   ├─ 调用 akshare 获取历史数据（从7年前至今）
    │   ├─ 存储到 stock_data 表
    │   └─ 进度：30% → 70%（批量处理）
    └─ 进度：70%
    ↓
【步骤5】计算技术指标
    ├─ 遍历所有股票的历史数据
    ├─ 对每只股票：
    │   ├─ 计算 MA5、MA10、MA20、MA60
    │   ├─ 计算 MACD、KDJ、RSI、BOLL、OBV
    │   ├─ 更新到 stock_data 表
    │   └─ 进度：70% → 90%
    └─ 进度：90%
    ↓
【步骤6】识别板块龙头股
    ├─ 对每个板块：
    │   ├─ 统计板块内各股票的市值和成交量
    │   ├─ 计算综合评分
    │   ├─ 按评分排序，取前N只
    │   └─ 存储到 sector_leaders 表
    └─ 进度：95%
    ↓
【步骤7】初始化预测模型
    ├─ 使用历史数据训练基础模型
    ├─ 保存模型参数
    └─ 进度：100%
    ↓
显示：数据更新完成！
刷新首页显示
```

### 2.2 增量数据更新（每日）

```
用户点击"更新数据"
    ↓
进度提示：正在更新数据...
    ↓
【步骤1】检查最后更新日期
    ├─ 从数据库读取最后交易日
    ├─ 获取当前最新交易日
    └─ 确定需要更新的日期范围
    ↓
【步骤2】获取最新交易日数据
    ├─ 对每只股票：
    │   ├─ 调用 akshare 获取最新日K数据
    │   ├─ 插入到 stock_data 表
    │   └─ 进度：30%
    ↓
【步骤3】更新技术指标
    ├─ 对更新了数据的股票：
    │   ├─ 重新计算所有技术指标
    │   ├─ 更新到 stock_data 表
    │   └─ 进度：60%
    ↓
【步骤4】更新板块龙头股
    ├─ 重新计算各板块龙头股
    ├─ 更新 sector_leaders 表
    └─ 进度：80%
    ↓
【步骤5】运行预测模型
    ├─ 对板块龙头股：
    │   ├─ 运行短期预测模型
    │   ├─ 运行中期预测模型
    │   ├─ 运行长期预测模型
    │   └─ 存储到 predictions 表
    └─ 进度：100%
    ↓
显示：数据更新完成！
刷新首页显示
```

## 3. 用户操作流程

### 3.1 查看板块总览流程

```
用户打开应用
    ↓
显示首页（板块总览）
    ├─ 左侧：板块选择下拉框（默认选中第一个板块）
    ├─ 中间：板块龙头股表格
    └─ 右侧：板块整体趋势K线图
    ↓
用户切换板块
    ├─ 点击板块选择下拉框
    ├─ 选择其他板块
    ↓
系统重新加载
    ├─ 查询新板块的龙头股列表
    ├─ 查询新板块的趋势数据
    ├─ 更新龙头股表格
    └─ 更新K线图
    ↓
完成，显示新板块信息
```

### 3.2 查看股票详情流程

```
用户在首页点击某只股票
    ↓
导航到股票详情页
    ↓
【左侧】加载图表
    ├─ 加载该股票的历史K线数据
    ├─ 加载该股票的技术指标数据
    ├─ 绘制K线图（默认显示6个月）
    ├─ 绘制成交量图
    ├─ 绘制移动平均线
    └─ 默认显示MACD指标
    ↓
【右侧】加载预测
    ├─ 从 predictions 表读取该股票的预测结果
    ├─ 读取短期预测
    ├─ 读取中期预测
    ├─ 读取长期预测
    └─ 计算综合建议
    ↓
【右侧】显示信息
    ├─ 显示股票基本信息卡片
    ├─ 显示短期预测卡片
    ├─ 显示中期预测卡片
    ├─ 显示长期预测卡片
    └─ 显示综合建议卡片
    ↓
完成，显示股票详情
```

### 3.3 切换技术指标流程

```
用户在股票详情页点击技术指标标签页
    ├─ 当前显示：MACD
    ├─ 用户点击：KDJ
    ↓
切换显示内容
    ├─ 隐藏 MACD 指标图
    ├─ 显示 KDJ 指标图
    └─ 标签页高亮 KDJ
    ↓
完成
```

### 3.4 切换K线时间范围流程

```
用户在股票详情页点击时间范围选择器
    ├─ 当前显示：6个月
    ├─ 用户点击：1年
    ↓
重新加载数据
    ├─ 查询最近1年的K线数据
    ├─ 重新绘制K线图
    └─ 更新时间范围选择器状态
    ↓
完成
```

## 4. 预测计算流程

### 4.1 单只股票预测流程

```
触发预测计算
    ├─ 数据更新后自动触发
    └─ 或用户手动触发
    ↓
加载股票数据
    ├─ 从 stock_data 表读取该股票的历史数据
    ├─ 读取技术指标
    ├─ 读取基本面数据
    └─ 组织成特征向量
    ↓
【短期预测】
    ├─ 提取短期特征（最近20天数据）
    ├─ 运行短期预测模型
    ├─ 输出：信号（买入/卖出/持有）+ 置信度
    ├─ 生成推理依据
    └─ 存储到 predictions 表（horizon='short'）
    ↓
【中期预测】
    ├─ 提取中期特征（最近120天数据）
    ├─ 运行中期预测模型
    ├─ 输出：信号（买入/卖出/持有）+ 置信度
    ├─ 生成推理依据
    └─ 存储到 predictions 表（horizon='medium'）
    ↓
【长期预测】
    ├─ 提取长期特征（全部历史数据）
    ├─ 运行长期预测模型
    ├─ 输出：信号（买入/卖出/持有）+ 置信度
    ├─ 生成推理依据
    └─ 存储到 predictions 表（horizon='long'）
    ↓
【生成综合建议】
    ├─ 读取三个周期的预测结果
    ├─ 按权重计算：
    │   ├─ 短期权重：30%
    │   ├─ 中期权重：40%
    │   └─ 长期权重：30%
    ├─ 生成综合信号和置信度
    └─ 返回预测结果
    ↓
完成
```

### 4.2 批量预测流程

```
用户更新数据后
    ↓
获取需要预测的股票列表
    ├─ 从 sector_leaders 表读取所有板块龙头股
    └─ 去重得到股票列表
    ↓
遍历股票列表
    ↓
对每只股票执行单只股票预测流程
    ↓
更新进度条
    ├─ 显示：正在预测 xxx (N/M)
    └─ 计算进度百分比
    ↓
全部完成后
    ↓
显示：预测计算完成！
刷新界面显示
```

## 7. 错误处理流程

### 7.1 数据更新失败处理

```
数据更新过程中出现错误
    ↓
捕获异常
    ├─ 网络错误：akshare API调用失败
    ├─ 数据错误：返回数据格式不正确
    ├─ 存储错误：数据库写入失败
    └─ 其他错误
    ↓
显示错误提示
    ├─ 错误类型：xxx
    ├─ 错误信息：xxx
    └─ 建议操作：重试/稍后再试/检查网络
    ↓
处理选项
    ├─ 用户点击"重试" → 重新执行数据更新
    ├─ 用户点击"取消" → 取消更新，恢复之前状态
    └─ 用户点击"忽略" → 跳过错误，继续后续步骤
    ↓
根据用户选择处理
```

### 7.2 预测计算失败处理

```
单只股票预测计算失败
    ↓
捕获异常
    ↓
记录错误日志
    ├─ 股票代码：xxx
    ├─ 错误时间：xxx
    ├─ 错误类型：xxx
    └─ 错误详情：xxx
    ↓
在界面显示警告
    ├─ 提示：股票 xxx 预测失败
    ├─ 原因：xxx
    └─ 该股票暂时无法提供预测
    ↓
继续处理其他股票
    ↓
全部完成后
    ↓
显示：预测计算完成！失败N只股票
```

## 10. 关键路径说明

### 10.1 首次使用关键路径
1. 启动应用 → 创建数据库
2. 点击"完整数据更新"
3. 等待数据获取完成（可能较长时间）
4. 首页显示板块信息
5. 点击股票查看详情

### 10.2 日常使用关键路径
1. 启动应用
2. 点击"更新数据"
3. 等待更新完成
4. 查看板块龙头股
5. 点击感兴趣的股票
6. 查看预测建议
7. 做出投资决策

### 10.3 性能优化关键点
1. 数据获取：使用批量请求，减少API调用次数
2. 指标计算：使用向量化计算，提高效率
3. 预测计算：预测结果缓存，避免重复计算
4. 界面渲染：使用异步加载，提高响应速度
