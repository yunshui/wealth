# PRD - 产品需求文档

## 1. 项目概述

### 1.1 项目名称
人机协同A股智能投资决策系统

### 1.2 项目目标
构建一个本地桌面应用，基于7年A股历史数据，通过AI模型对板块龙头股进行短中长期预测，为个人投资者提供买卖操作建议，最终由用户决策是否执行。

### 1.3 目标用户
- 个人股票投资者
- 具备基本股票知识但希望借助AI辅助决策的用户
- 倾向于本地化、数据隐私保护的用户

---

## 2. 核心功能需求

### 2.1 数据管理

#### 2.1.1 数据源
- **数据提供方**：使用系统内置的免费数据源 (akshare)
- **数据类型**：
  - A股历史价格数据（开高低收、成交量、成交额）
  - 板块分类数据（行业板块、概念板块）
  - 股票基本信息（股票代码、名称、所属行业/板块）
  - 财务数据（市值、市盈率等，akshare提供）
- **历史跨度**：7年历史数据
- **更新频率**：每日收盘后更新一次（用户手动触发）
- **数据存储**：本地SQLite数据库

#### 2.1.2 板块龙头股识别
- **覆盖范围**：先覆盖主要板块试点（如：科技、医药、消费、金融、制造等）
- **筛选标准**：
  - 市值：板块内市值排名前列
  - 成交量：近期活跃度高
  - 综合评分：市值 × 权重1 + 成交量 × 权重2 + 其他指标
- **识别方式**：系统自动识别各板块龙头股，无需用户手动指定

#### 2.1.3 技术指标计算
- **基础指标**：
  - 移动平均线：MA5、MA10、MA20、MA60
  - 成交量指标：VOL
- **趋势指标**：
  - MACD（DIF、DEA、MACD柱）
  - BOLL（布林带上中下轨）
- **震荡指标**：
  - RSI（相对强弱指标，6日、12日、24日）
  - KDJ（K、D、J值）
- **动量指标**：
  - OBV（能量潮）
- 所有指标自动计算并存储，每日更新时重新计算

### 2.2 预测功能

#### 2.2.1 三周期预测
系统对每只股票分别进行三个时间周期的预测：

**短期预测（1-5天）**
- 目标：预测未来1-5个交易日的涨跌方向
- 特征：技术指标主导，量价关系分析
- 输出：买入/卖出/持有 + 置信度百分比（0-100%）

**中期预测（1-3个月）**
- 目标：预测未来1-3个月的趋势方向
- 特征：趋势判断 + 技术面 + 基本面辅助
- 输出：买入/卖出/持有 + 置信度百分比（0-100%）

**长期预测（3个月以上）**
- 目标：预测3个月以上的长期趋势和价值
- 特征：价值分析 + 行业周期 + 长期趋势
- 输出：买入/卖出/持有 + 置信度百分比（0-100%）

#### 2.2.2 预测信号定义
- **买入**：看好股票上涨，建议考虑买入
- **卖出**：看好股票下跌，建议考虑卖出
- **持有**：趋势不明确或处于震荡，建议持有观望

#### 2.2.3 置信度
- 范围：0% - 100%
- 含义：模型对该预测的可信程度
- 展示方式：进度条 + 数字显示

#### 2.2.4 推理依据
每个预测必须提供可解释的推理依据，包括：
- 关键技术指标的当前状态和变化
- 量价关系的分析
- 趋势方向的判断
- 其他相关因素的说明

### 2.3 决策支持

#### 2.3.1 人机协同模式
**系统职责**：
- 给出明确的买入/卖出/持有建议
- 提供详细的预测结果（短中长期）
- 展示置信度和推理依据
- 提供数据可视化支持

**用户职责**：
- 查看系统提供的建议和数据
- 结合自己的判断做出最终决策
- 决定是否执行买卖操作

**交互模式**：
- 系统提供建议，用户自主决策
- 系统不执行任何交易操作
- 用户可以采纳也可以拒绝系统建议

#### 2.3.2 综合建议生成
系统汇总短中长期三个周期的预测结果，生成最终的综合买卖建议：

**生成规则**：
- 如果三个周期方向一致（如：都建议买入），综合建议为该方向
- 如果存在分歧，根据置信度加权计算
- 短期预测权重：30%
- 中期预测权重：40%
- 长期预测权重：30%

**综合建议展示**：
- 明确的买入/卖出/持有建议
- 综合置信度
- 各周期预测结果的摘要

### 2.4 界面功能

#### 2.4.1 首页 - 板块总览
**功能描述**：显示各个板块的整体情况和龙头股列表

**界面元素**：
- 板块选择器：下拉选择器，列出所有主要板块
- 板块列表表格：
  - 板块名称
  - 板块内龙头股数量
  - 板块整体趋势（上涨/下跌/震荡）
- 板块龙头股表格（选中板块后显示）：
  - 股票代码
  - 股票名称
  - 最新价格
  - 涨跌幅
  - 短期预测信号
  - 中期预测信号
  - 长期预测信号
  - 综合建议
- 板块整体趋势K线图（可切换日线/周线）

#### 2.4.2 股票详情页
**功能描述**：展示单只股票的详细分析和预测建议

**界面布局**：

**左侧区域（约60%宽度） - 图表区**：
- 主K线图：
  - 时间范围选择器（1月/3月/6月/1年/全部）
  - K线图（开高低收）
  - 成交量柱状图
  - 移动平均线叠加（MA5、MA10、MA20）
- 技术指标图表（可切换）：
  - MACD指标
  - KDJ指标
  - RSI指标
  - BOLL指标

**右侧区域（约40%宽度） - 预测建议区**：
- 股票基本信息卡片：
  - 股票代码和名称
  - 所属板块
  - 最新价格
  - 今日涨跌幅
  - 市值、市盈率等关键数据

- 预测建议卡片（三个卡片，分别对应短中长期）：
  - **短期预测卡片**：
    - 标题：短期预测（1-5天）
    - 信号：买入/卖出/持有（用颜色标识：绿涨红跌灰持）
    - 置信度：进度条 + 百分比数值
    - 推理依据：3-5条关键理由，每条一行
  - **中期预测卡片**：同上结构
  - **长期预测卡片**：同上结构

- 综合建议卡片：
  - 标题：综合建议
  - 信号：买入/卖出/持有（大字突出显示）
  - 综合置信度
  - 各周期预测汇总：短期xxx，中期xxx，长期xxx → 建议xxx
  - 操作提示：如"强烈建议关注"、"建议减仓"等

#### 2.4.3 功能菜单
**菜单项**：
1. **更新数据**
   - 功能：手动触发数据更新
   - 作用：获取最新交易日数据，重新计算指标和预测
   - 状态：显示更新进度和最后更新时间
   - 按钮样式："立即更新"（灰色时表示正在更新中）

2. **重新训练模型**
   - 功能：使用最新数据重新训练预测模型
   - 作用：优化模型准确性
   - 执行方式：后台运行，显示训练进度
   - 使用场景：累计一定时间后（如每月）手动触发

3. **历史预测回顾**
   - 功能：查看过往预测的准确率统计
   - 内容：
     - 各周期预测的准确率（短期/中期/长期）
     - 不同信号类型的准确率（买入/卖出/持有）
     - 时间维度：近1月/3月/6月/全部
   - 展示形式：表格 + 简单图表

4. **系统设置**（可选）
   - 数据源切换
   - 板块管理
   - 界面主题切换

## 3. 非功能需求

### 3.1 性能要求
- 数据更新：每日数据获取和处理应在5分钟内完成
- 预测计算：单只股票的三周期预测应在3秒内完成
- 界面响应：页面切换应在2秒内完成
- 首次加载：首页加载应在5秒内完成

### 3.2 易用性要求
- 界面简洁直观，无需专业培训即可使用
- 关键信息一目了然
- 颜色标识符合中国股市习惯（红涨绿跌）
- 提供必要的操作提示和说明

### 3.3 可靠性要求
- 数据更新失败时有明确错误提示和重试机制
- 预测计算失败时显示错误但不影响其他功能
- 数据存储采用事务机制，避免数据不一致

### 3.4 本地化要求
- 所有数据和计算均在本地完成
- 数据存储在本地SQLite数据库
- 不需要联网即可查看历史数据
- 数据更新时才需要联网

---

## 4. 数据隐私和安全

### 4.1 数据隐私
- 用户投资决策数据仅存储在本地
- 不上传任何用户数据到云端
- 不收集用户行为数据

### 4.2 数据安全
- SQLite数据库文件加密存储（可选）
- 定期自动备份（可选）

## 5. 成功标准

### 5.1 功能完整性
- 能够成功获取A股历史数据和实时数据
- 能够正确识别各板块龙头股
- 能够计算各类技术指标
- 能够进行短中长期预测
- 能够生成综合买卖建议
- 界面功能完整可用

### 5.2 预测准确性
- 短期预测准确率目标：>55%

- 长期预测准确率目标：>50%

- 界面操作流畅，无卡顿
- 错误提示清晰明确
- 用户能够理解预测依据

## 6. 限制和约束

### 6.1 技术约束
- 使用免费数据源（akshare），数据源稳定性依赖第三方
- 预测模型基于历史数据，不保证未来表现
- 本地运行，硬件资源有限

### 6.2 免责声明
- 系统仅供辅助决策，不构成投资建议
- 用户需自行承担投资决策风险
- 开发者不对投资损失负责
- 股市投资有风险，入市需谨慎
