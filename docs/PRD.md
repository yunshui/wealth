# PRD - 产品需求文档

## 1. 项目概述

### 1.1 项目名称
人机协同A股智能投资决策系统

### 1.2 项目目标
构建一个本地桌面应用，基于7年A股历史数据，通过AI模型对板块龙头股进行短中长期预测，为个人投资者提供买卖操作建议，最终由用户决策是否执行。

### 1.3 目标用户
- 个人股票投资者
- 具备基本股票知识但希望借助AI辅助决策的用户
- 倾向于本地化、数据隐私保护的用户

## 2. 核心功能需求

### 2.1 数据管理

#### 2.1.1 数据源
- **数据提供方**：使用系统内置的免费数据源 (akshare)
- **数据类型**：
  - A股历史价格数据（开高低收、成交量、成交额）
  - 板块分类数据（行业板块、概念板块）
  - 股票基本信息（股票代码、名称、所属行业/板块）
  - 财务数据（市值、市盈率等，akshare提供）
- **历史跨度**：7年历史数据
- **更新频率**：每日收盘后更新一次（用户手动触发）
- **数据存储**：本地SQLite数据库

#### 2.1.2 板块龙头股识别
- **覆盖范围**：聚焦关键板块，覆盖主要投资方向
  - 行业板块（10个）：银行、保险、房地产开发、白酒、生物制品、医疗服务、半导体、消费电子、电力设备、电池、光伏设备
  - 概念板块（10个）：AI芯片、ChatGPT概念、一带一路、专精特新、中特估、储能概念、充电桩、数字经济、汽车整车、白酒（概念）
- **筛选标准**：
  - 市值：板块内市值排名前列
  - 成交量：近期活跃度高
  - 综合评分：市值 × 权重1 + 成交量 × 权重2 + 其他指标
- **识别方式**：系统自动识别各板块龙头股，无需用户手动指定
- **数据管理**：所有板块和龙头股数据存储在SQLite数据库中

#### 2.1.3 技术指标计算
- **基础指标**：
  - 移动平均线：MA5、MA10、MA20、MA60
  - 成交量指标：VOL
- **趋势指标**：
  - MACD（DIF、DEA、MACD柱）
  - BOLL（布林带上中下轨）
- **震荡指标**：
  - RSI（相对强弱指标，6日、12日、24日）
  - KDJ（K、D、J值）
- **动量指标**：
  - OBV（能量潮）
- 所有指标自动计算并存储，每日更新时重新计算

### 2.2 预测功能

#### 2.2.1 三周期预测
系统对每只股票分别进行三个时间周期的预测：

**短期预测（1-5天）**
- 目标：预测未来1-5个交易日的涨跌方向
- 特征：技术指标主导，量价关系分析
- 输出：买入/卖出/持有 + 置信度百分比（0-100%）

**中期预测（1-3个月）**
- 目标：预测未来1-3个月的趋势方向
- 特征：趋势判断 + 技术面 + 基本面辅助
- 输出：买入/卖出/持有 + 置信度百分比（0-100%）

**长期预测（3个月以上）**
- 目标：预测3个月以上的长期趋势和价值
- 特征：价值分析 + 行业周期 + 长期趋势
- 输出：买入/卖出/持有 + 置信度百分比（0-100%）

#### 2.2.2 预测信号定义
- **买入**：看好股票上涨，建议考虑买入
- **卖出**：看好股票下跌，建议考虑卖出
- **持有**：趋势不明确或处于震荡，建议持有观望

#### 2.2.3 置信度
- 范围：0% - 100%
- 含义：模型对该预测的可信程度
- 展示方式：进度条 + 数字显示

#### 2.2.4 推理依据
每个预测必须提供可解释的推理依据，包括：
- 关键技术指标的当前状态和变化
- 量价关系的分析
- 趋势方向的判断
- 其他相关因素的说明

### 2.3 决策支持

#### 2.3.1 人机协同模式
**系统职责**：
- 给出明确的买入/卖出/持有建议
- 提供详细的预测结果（短中长期）
- 展示置信度和推理依据
- 提供数据可视化支持

**用户职责**：
- 查看系统提供的建议和数据
- 结合自己的判断做出最终决策
- 决定是否执行买卖操作

**交互模式**：
- 系统提供建议，用户自主决策
- 系统不执行任何交易操作
- 用户可以采纳也可以拒绝系统建议

#### 2.3.2 综合建议生成
系统汇总短中长期三个周期的预测结果，生成最终的综合买卖建议：

**生成规则**：
- 如果三个周期方向一致（如：都建议买入），综合建议为该方向
- 如果存在分歧，根据置信度加权计算
- 短期预测权重：30%
- 中期预测权重：40%
- 长期预测权重：30%

**综合建议展示**：
- 明确的买入/卖出/持有建议
- 综合置信度
- 各周期预测结果的摘要

**第三层：板块网格（主内容区）**
- 显示方式：横向多行网格（每行4个板块）

  - 概念板块（10个）：AI芯片、ChatGPT概念、一带一路、专精特新、中特估、储能概念、充电桩、数字经济、汽车整车、白酒（概念）

#### 2.4.4 数据更新模块
**功能描述**：手动触发数据更新，支持数据库驱动的并行处理

**界面元素**：
- 数据库状态：
  - 已初始化/未初始化
  - 最后更新日期
  - 初始化按钮（如需要）

- 更新操作按钮（三列并排）：
  1. 更新板块数据
     - 从akshare API获取板块信息
     - 保存板块到数据库
     - 识别并保存龙头股

  2. 更新股票数据
     - 从数据库读取板块列表
     - 并行获取股票历史数据（板块级8线程 + 股票级16线程）
     - 自动计算技术指标
     - 智能跳过最近3天的数据
     - 实时进度显示：板块进度、已处理股票数、跳过股票数

  3. 更新技术指标
     - 并行重新计算所有股票的技术指标
     - 适用于指标算法更新场景

**性能指标**：
- 数据更新：20个板块的股票数据应在5分钟内完成（优化后）
- 预测计算：单只股票的三周期预测应在3秒内完成
- 界面响应：页面切换应在2秒内完成
- 首次加载：首页加载应在5秒内完成

#### 2.4.5 数据库驱动架构

**数据存储**：
- 板块列表：存储在 SQLite 数据库的 `sectors` 表
- 龙头股信息：存储在 SQLite 数据库的 `sector_leaders` 表
- 更新配置：硬编码在代码中（可后续调整为配置项）

**优势**：
- 单一数据源，无需维护配置文件
- 数据更新时直接更新数据库
- 支持动态板块管理

## 3. 非功能需求

### 3.1 性能要求
- 数据更新：每日数据获取和处理应在5分钟内完成（优化后目标，20个板块）
- 预测计算：单只股票的三周期预测应在3秒内完成
- 界面响应：页面切换应在2秒内完成
- 首次加载：首页加载应在5秒内完成
- 并行处理：
  - 板块级并行：8个线程同时处理不同板块
  - 股票级并行：16个线程同时处理不同股票
  - 数据跳过机制：自动跳过最近3天的数据，减少重复下载

### 3.2 易用性要求
- 四层UI架构，清晰的视觉层次
- 左侧导航，右侧内容，操作直观
- 板块网格展示，关键指标一目了然
- 颜色标识符合中国股市习惯（红涨绿跌）
- 实时进度显示，更新状态清晰
- 提供必要的操作提示和说明

### 3.3 可靠性要求
- 数据库作为单一真实来源（Single Source of Truth）
- 数据更新失败时有明确错误提示和重试机制
- 预测计算失败时显示错误但不影响其他功能
- 数据存储采用事务机制，避免数据不一致
- 并行处理使用线程安全机制，防止数据竞争
- 每个线程使用独立的数据库连接，避免锁竞争

### 3.4 本地化要求
- 所有数据和计算均在本地完成
- 数据存储在本地SQLite数据库
- 板块配置和龙头股信息保存在配置文件
- 不需要联网即可查看历史数据
- 数据更新时才需要联网
